What Changed

  • Joined tools: `get_student_overview`, `get_student_teachers`, `get_student_location`, `find_students_same_subjects`, and `get_level_teachers` now cover the benchmark’s hard multi-file reasoning directly.
  • One-shot answer flow: once a joined tool returns, the harness immediately forces a final answer instead of letting the local model loop.
  • Cleaner payloads: joined tools now return explicit summary fields that the model can quote safely without recomputing the facts.
  • Generic tool improvements: `query` now supports `in` and `not_in`, and CSV reads are cached within a run.
  • Scorer fixes: compound facts such as profile fields and `Subject (Rigour): Teacher` now score correctly, and the ambiguous Q14 interpretation is explicitly accepted.

Verification Scope

The report is intentionally scoped to the current optimised tools mode, because that is the configuration that was revalidated after the tooling redesign.

The previous baseline and no-tools figures were not rerun for this refresh, so they are omitted rather than carried forward as stale numbers.
